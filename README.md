# cmpe272finalquestion10
respository for cmpe272 final involving approaches for secure file transfer

## Installation instructions
- Run `pip install -r requirements.txt` in a command terminal for both approach_a_direct and approach_b_broker directories

## Generating a 4GB file
- Run `python -c "open('test_4gb.bin','wb').write(b'\\0'*2*1024*1024*1024)"` in a command terminal to generate a 4GB file to test file transfer

## Running procedure (Approach A)
- Steps assume that you are in the `approach_a_direct` folder
1. Run `python receiver.py` through a command window to intially generate the public/private keys
2. Run `python sender.py --receiver-pubkey <public key file> <transfer file>` through a separate command window to prepare file transfer
- `<public key file>` refers to the `receiver_public.pem` file generated by the receiver
- `<transfer file>` refers to the file to transfer, use the 4GB file
3. Run `python receiver.py [--output-dir <output directory>]` through a command window while sender is active to start the file transfer
-`--output-dir` is optional and refers to where to put the output directory; outputs to `received_files` by default

## Running procedure (Approach B)
- Steps assume that you are in the `approach_b_direct` folder
1. Run `python generate_keys.py` to initially generate the public/private keys
2. Run `python broker.py` through a command window to set up the intermediary broker server
3. Run `python sender.py [--public-key <public key file>] <transfer file>` through a command window to transfer the file to the intermediary broker
- `--public-key` is optional and refers to the `receiver_public.pem` file location
- The broker should respond with a unique file_id for the sent file to be used with the receiver
4. Run `python receiver.py [--private-key <private key file>] <file_id> <output filename>` through a command window to transfer the file from the broker to the receiver
- `--private-key` is optional and refers to the `receiver_private.pem` file location
- `<file_id>` refers to the unique file_id for the corresponding file created by the broker
- `<output filename>` refers to the output filename of the file after transferring from the broker to the receiver

## Program notes
- The broker approach currently does not work with large file types compared to the direct approach
- -For some reason, the file transfer between the broker and the receiver immediately breaks if the file size is larger than 1GB
- -However, the file transfer between the sender and the broker works regardless of how large the file is
- -The file transfer between the sender and broker and receiver works for small file sizes
- For approach A, `sender.py` and `receiver.py` run on `localhost:8080` and `localhost:8081` respectively, while approach B has `broker.py` run on `localhost:8080`; approaches must be run separately
- Video demo link: https://youtu.be/rshf8-N5dO4
- AI agents used include Claude and Gemini; notes for their usage are shown in `AI_NOTES.md`
- Individual prompts for each approach are shown in `prompts.md` file for each approach
